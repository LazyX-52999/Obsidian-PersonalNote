设计模式1：
	学了两个设计模式，主要是简单熟悉一下，后期还会深度学习设计模式
	单例设计模式：
		我的理解：单例就是单个实例，也就是说单例设计模式只能有一个对象实例
		单例设计模式的两种方式：饿汉式 和 懒汉式
		饿汉式：
			步骤：
			1.将构造器私有化 （防止用户直接创建对象）
			2.在类的内部创建一个 static 的对象实例
			3.向外暴露一个公共的静态方法，返回对象实例
			4.代码实现
```java
class A {
	private String name;
	private A(String name) {
		this.name = name;
	}
	private static A a = new A("jack");
	public static A show() {
		return a;
	}
}
```
.
			饿汉式的弊端：可能创建了对象但是没有使用，造成资源浪费
		懒汉式：
			步骤：
				1.将构造器私有化 （防止用户直接创建对象）
				2.定义一个静态属性对象
				3.提供一个公共的静态方法，返回对象实例
				4.代码实现
```java
class A {
	private String name;
	private A(String name) {
		this.name = name;
	}
	private static A a;
	public static A show() {
		if(a == null) {
			a = new A("jack");
		}
		return a;
	}
}
```
.
			懒汉式弥补了饿汉式的弊端，只有用户使用 提供的公共的静态方法 时，才会返回对象实例，后面再次调用时，会返回上次创建的对象实例，从而保证了单例
		饿汉式 VS 懒汉式
			1.二者创建对象的时机不同
				饿汉式：在类加载的时候创建对象
				懒汉式：在使用的时候创建对象
			2.饿汉式没有线程安全问题，懒汉式有线程安全问题
			3.饿汉式可能会浪费资源(因为只要类加载了就会创建对象)

final关键字：
	final 可以修饰 类、属性、方法、局部变量
	final修饰类：
		==final class A { }==   A类不能被继承，但是可以创建对象
	final修饰属性：
		==private final static int N = 10;==   N通常被称为常量，用大写字母表示(X / XX_XX)
		常量必须要赋初值且不能被修改
		普通常量赋初值的三种方式：1.定义时   2.在构造器中   3.在代码块中
		静态常量赋初值的两种方式：1.定义时   2.在静态代码块中
	final修饰局部变量：
		==final int N = 10;==   N与final修饰属性差不多
	final修饰方法：
		==public final static void show() { }==   show方法不能被重写
	final不能修饰构造器
	final 和 static 往往搭配使用，效率更高，不会导致类的加载

抽象类：
	抽象类中学了 ==抽象类== 、==抽象方法== 。 是用 ==abstract== 修饰的 （只能修饰类和方法）
	抽象类：（本质还是个类）
		==abstract class A { }==   抽象类中一般存放 抽象方法、属性、非抽象方法、构造器、静态属性等(即：抽象类可以有任意成员)
		其作用是设计，具体实现是让 子类继承它 来 实现抽象类中*所有* 抽象方法
		抽象类不能被实例化
		当一个类中有了抽象方法，那么这个类一定是抽象类
	抽象方法：
		==public abstract void show();==   在抽象类中可以省略 public abstract
		抽象方法不能有主体
		抽象方法不能用 private final static 来修饰

设计模式2：
	模板设计模式：
		模板设计模式是抽象类的最佳实践
		在一个抽象类中写一个方法，这个方法中的内容是抽象类的所有子类中重复的内容和抽象方法；抽象类中有抽象方法，这个抽象方法是由抽象类的子类来实现的
```java
public class TestTemplate {  
    public static void main(String[] args) {  
        AA aa = new AA();  
        aa.calculateTime();  
  
        BB bb = new BB();  
        bb.calculateTime();  
    }  
}  
  
abstract class Template {  
  
    public abstract void job();  
  
    public void calculateTime() {  
        long start  = System.currentTimeMillis();  
        job();  
        long end = System.currentTimeMillis();  
        System.out.println("程序执行了" + (end - start));  
    }  
}  
  
class AA extends Template {  
    long num = 0;  
    @Override  
    public void job() {  
        for (long i = 1; i <= 1000000; i++) {  
            num += i;  
        }  
    }  
}  
  
class BB extends Template {  
    long num = 0;  
    @Override  
    public void job() {  
        for (long i = 1; i <= 5000000; i++) {  
            num += i;  
        }  
    }  
}
```

接口：
	我的理解：
		接口是用来提供一些没有实现的方法(即：抽象方法)，当某个类要使用的这个方法的时候，就会调用这个接口中的抽象方法并写出具体的方法主体。
```java
interface IA {
	public final static int N = 10; //属性
	int N = 10; //与上面的属性定义是等价的，可以不写 public final static
	public abstract void show1(); //抽象方法
	void show1(); //与上面的方法定义是等价的，可以不写 public abstract
	public static void show2() { //jdk8及以后可以有静态方法
		System.out.println("show2方法");	
	}
	default public void show3() { //jdk8及以后可以有默认实现方法
		System.out.println("show3方法");	
	}
}

class A implements IA {
	private int a = 20;
	@override
	void show1() {
		System.out.println("show1方法");	
	}
	public void show4() {
		System.out.println("show4方法");	
	}
}
```
.             jdk7及以前：接口中只能有抽象方法
		jdk8及以后：接口可以有抽象方法，静态方法，默认方法，属性（即：接口中可以有方法的具体的实现）
	接口的作用：
		可以统一规范（方法名），方便管理。一般是由产品经理来定义的接口，程序员根据接口中的内容完成相应的业务逻辑，因为不止有一个程序员，所以每个程序员写的代码的规范是不一样的，所以就要有接口这个类来统一规范，使代码更方便管理
	接口的细节：
		1.接口不能被实例化，也就是说接口不能像普通类一样来创建对象
		2.接口中的所有方法都是 public ，接口中的抽象方法可以不用abstract修饰
		3.一个普通类实现接口，要实现接口中所有的抽象方法
		4.抽象类实现接口，可以不用实现接口的方法
		5.一个类可以实现多个接口
```java
class Pig implements IB,IC { }
```
.		     6.接口中的属性只能使final，只能用 public static final 来修饰
		7.接口中属性访问形式：接口名.属性名
		8.接口的修饰符只能是 public 和 默认
		9.接口不能继承其他的类，但是接口可以继承多个别的接口
```java
interface A {}
interface B {}
interface C extends A,B {}
```

接口 VS 继承
	java提供的*实现*这个机制是对*java单继承机制*的补充
	接口和继承解决的问题不同：
		继承的价值：解决代码的==复用性==和==可维护性==
		接口的价值：设计好各种规范（方法），让其他类去实现这些方法。更加灵活
	接口比继承更加灵活：
		继承是满足 ==is - a== 的关系
		接口是满足 ==like - a== 的关系
	接口在一定程度上实现代码解耦（接口规范性 + 动态绑定机制）

接口的多态：
```java
public class Interface02 {  
    public static void main(String[] args) {  
        A1 a1 = new A1();  
        A2 a2 = new A2();  
        A3 a3 = new A3();  
        Interface02 interface02 = new Interface02();  
        interface02.show(a1);  
        interface02.show(a2);  
        System.out.println(A1.n1);  
        System.out.println(A2.n2);  
        interface02.show(a3);  
        interface02.show1(a3);

    public void show(USB usb) {  
        usb.start();  
        usb.stop();  
    }  
    public void show1(USB1 usb1 ) {

interface USB2 extends USB {
```

接口的多态传递：
```java
public class InterfacePolyPass {
	public static void main(String[] args) {
		IB ib = new C();
		//IB 继承了 IA ，C 实现了 IB 接口
		//那么 C 也实现了 IA 接口
		//这就是接口的多态传递
		IA ia = new C();
	}
}

interface IA {
	void hi();
}
interface IB extends IA {}
class C implements IB {
	@Override
	public void hi() {
		System.out.println("hi方法");
	}
}
```
