#### JavaSE
今日学习了 ==super== 、 ==方法重写== 、 ==多态==
super：
	我的理解是：
		在继承中需要用到super，可以在子类中访问父类，可以在子类的构造器中访问父类的构造器（当父类中没有有参构造器时，默认在子类的构造器的第一行有 super(); 语句（可有可无））
	当类的继承链很多时，super遵循就近原则，同时也遵循访问权限的规则
	主要的用法就是下面的表格中的内容：（与 this 作了类比）

|  区别点  |            this             |         super         |
| :---: | :-------------------------: | :-------------------: |
| 访问属性  | 访问本类中的属性，如果本类没有此属性则从父类中继续查找 |       从父类开始查找属性       |
| 调用方法  | 访问本类中的方法，如果本类没有此方法则从父类中继续查找 |       从父类开始查找方法       |
| 调用构造器 |     调用本类构造器，必须放在构造器的首行      | 调用父类的构造器，必须放在子类构造器的首行 |
|  特殊   |           表示当前对象            |       子类中访问父类对象       |
	==（注：在同一个构造器中， this 和 super 不能共存）==

方法重写：
	我的理解是：
		在子类中可以有多个与父类的方法相同的方法，这就是方法重写
	具体的细节体现在下面的表格中：（与 方法重载 作了类比）

|      名称      | 发生范围 | 方法名  |      参数列表       |            返回类型            |  修饰符   |
| :----------: | :--: | :--: | :-------------: | :------------------------: | :----: |
| 重载(overload) |  本类  | 必须相同 | 类型、个数、顺序至少有一个不同 |            无要求             |  无要求   |
| 重写(override) | 父子类  | 必须相同 |       相同        | 相同/子类重写的方法的返回类型是父类的返回类型的子类 | 子类>=父类 |
	访问修饰符：public > protected > 默认 > private
	==（疑问：什么时候（哪些场景）会用到方法重写?）==
	

多态：
	我的理解是：
		多态就是一个对象的多种形态。多态是建立在 继承 和 封装 的基础上的。在一个项目中，会有很多个类，这些类中可能会有相同或不同的成员，我们可以使用同一种调用方式来使这些成员运行出多种形态（即：不同的运行结果）
	多态的好处：
		可以调高代码的复用性，让代码更容易维护
	方法的多态：
		方法的多态主要体现在 方法重载 和 方法重写 上。
		当一个子类中，有多种方法构成了方法重载，那么我们在调用方法的时候，就可以通过传入不同的参数来调用不同的方法
		当子类和父类中的方法构成了方法重写，那么我们就可以调用不同类中的方法
	==对象的多态==： 
		一个对象的 编译类型 和 运行类型 可以不一致。
		编译类型在定义对象的时候就确定了，不可以改变
		运行类型是可以改变的
		举个例子：Animal animal = new Dog();
			animal的编译类型 是 Animal，运行类型 是 Dog
		就是说，在定义的时候，编译类型看 ‘=’ 的左边，运行类型看 ‘=‘ 的右边
	向上转型：
		语法：父类类型 引用名 = new 子类类型();
		本质：父类的引用指向了子类的对象/将子类的对象赋给父类的引用
		特点：可以调用父类中的所有成员，不能调用子类中的特有成员
	向下转型：
		语法：子类类型 引用名 = (子类类型)  父类引用;
		本质：将父类的引用强制转换成子类类型（不能强转父类的对象）
		特点：父类的引用必须指向的是当前目标类型的对象
			   当向下转型后，可以调用子类类型中的所有成员
	（注：我的理解是“父类的引用”就是“它指向子类对象在堆中的内存地址”，这个地址会赋值给”父类的对象名“）
	属性重写的问题：
		属性没有重写之说，访问属性的值时，看编译类型
	instanceof比较操作符：
		用于判断对象的==运行类型==是否为 XX类型 或 XX类型的子类型
	==java的动态绑定机制==：
		当==调用方法==的时候(==看运行类型==)，该方法会和该对象的 内存地址/运行类型 绑定  
		当==调用属性==的时候(==看编译类型==)，没有动态绑定机制，哪里声明，哪里使用
	多态数组：
		数组的定义类型为父类类型
		数组中的元素类型是子类类型
		当要调用子类的特有方法时，可以通过 类型判断 + 向下转型 来调用
