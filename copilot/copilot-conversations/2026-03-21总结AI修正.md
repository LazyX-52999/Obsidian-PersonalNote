### 2. **方法重写（Override）的场景**
你提出了一个很好的问题：“**什么时候（哪些场景）会用到方法重写？**”
- **常见场景**：
    - **实现多态**：父类引用指向子类对象时，调用重写方法会执行子类逻辑（如 `Animal a = new Dog(); a.sound();`）。
    - **定制化行为**：子类需要修改或扩展父类方法的功能（例如，`JFrame` 子类重写 `paint()` 方法来自定义绘图）。
    - **模板方法模式**：父类定义算法骨架，子类重写其中某些步骤。
    - **接口/抽象类实现**：实现接口方法或抽象方法本身就是一种重写。

### 3. **多态部分的细节优化**
- **“父类的引用就是父类的对象名”** → 更准确的说法是：“**父类的引用变量**，它指向子类对象在堆中的内存地址。”

    ```java
    a.show(); // 输出"B"（方法看运行类型B）
    System.out.println(a.num); // 输出10（属性看编译类型A）
    ```

### 4. **instanceof的补充**

    - 例如：`animal instanceof Dog` 为 true 的条件是 `animal` 的运行类型是 `Dog` 或其子类。

### 5. **多态数组的实用场景**

**ai**: **一句话理解多态：**

**“同一个方法调用，因为对象不同，会产生不同的行为结果。”**

---

### 更通俗的解释：
就像你喊“吃饭了！”：
- 如果是**狗**，它会跑过来吃狗粮
- 如果是**猫**，它会慢悠悠走过来吃猫粮  
- 如果是**鱼**，它会在水里吃鱼食

你喊的是同一个指令“吃饭了”，但不同动物（对象）执行这个指令时，表现出的行为（方法实现）各不相同——这就是多态。

---

### 对应到你的笔记中：
```java
Animal animal = new Dog();  // 编译类型Animal，运行类型Dog
animal.eat();  // 调用的是Dog的eat()方法

animal = new Cat();  // 运行类型改为Cat
animal.eat();  // 现在调用的是Cat的eat()方法
```
**同一个 `animal.eat()` 调用，因为 `animal` 实际指向的对象不同（Dog或Cat），执行了不同的 `eat()` 方法实现。**

这就是多态的核心：**方法调用看运行类型，行为由实际对象决定**。
