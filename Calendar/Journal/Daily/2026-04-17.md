#### JavaSE
今日学习了 ==代码块==
我的理解：
	代码块是写在类中的，代码块中可以写本类中多个构造器中的语句，还可以写访问成员(核心目的是“初始化”和“执行通用逻辑”) 的语句。其可以降低代码的冗余度。
代码块的分类：
	静态代码块 和 普通代码块
	static {......}      {......}
代码块的执行：
	普通代码块：每创建一次对象，就会执行一次普通代码块中的语句(即:==可以执行多次==)
	静态代码块：当类加载的时候，==只会执行一次==静态代码块中的语句
类被加载的情况：
	1.创建对象实例时   new 类名();
	2.创建子类对象实例时（其父类也会被加载）
	3.访问类的静态成员时   类名.静态属性;   类名.静态方法;
调用优先级：
	静态 > 普通 > 构造器
	1.父类的静态代码块和静态属性（优先级一样，按定义顺序执行）
	2.子类的静态代码块和静态属性（优先级一样，按定义顺序执行）
	3.父类的普通代码块和普通属性初始化（优先级一样，按定义顺序执行）
	4.父类的构造方法
	5.子类的普通代码块和普通属性初始化（优先级一样，按定义顺序执行）
	6.子类的构造方法
[[JavaSE Easy-to-forget Key Points]]：
	在构造器中，默认会有 super 和 普通代码块 
```java
	public A {
		A() {
			super();
			{   };
			System.out.println("A构造器");
		}
	}
```
