## 设计模式-单例设计模式
只能创建一个对象就叫单例（单个实例）
	饿汉式：
		步骤：
			1.将构造器私有化 （防止用户直接创建对象）
			2.在类的内部创建一个 static 的对象实例
			3.向外暴露一个公共的静态方法，返回对象实例
			4.代码实现
		饿汉式的弊端：可能创建了对象但是没有使用
	懒汉式：
		步骤：
			1.将构造器私有化 （防止用户直接创建对象）
			2.定义一个静态属性对象
			3.提供一个公共的静态方法，返回对象实例
			4.代码实现
		懒汉式弥补了饿汉式的弊端，只有用户使用 提供的公共的静态方法 时，才会返回对象实例，后面再次调用时，会返回上次创建的对象实例，从而保证了单例
